# Логика расчётов и договорённости

Документ описывает правила, используемые при расчёте метрик и формировании отчётов.

---

# 1. Общие принципы

- Основная единица — дефект (issue)
- Источник данных — YouTrack
- Данные обогащаются:
  - версиями
  - связями с PS (support)
  - расчётными полями

---

# 2. Affected version

## Логика заполнения

1. Если есть связь с PS: Affected version = PS_Версия
2. Если нет:
- определяется по дате Created
- сравнивается с датами из листа Versions
- выбирается ближайшая версия вперёд

---

# 3. Fix version

## Логика

### 1. Если статус = Аннулирована
Fix version = пусто
### 2. Если есть Release и он != "Не определен"
Fix version = Release

### 3. Иначе:
- берём дату Resolved
- ищем ближайшую версию вперёд по дате
- присваиваем её

---

# 4. Lifetime
Lifetime = Resolved - Created (в рабочих днях)

Если Resolved пустой → Lifetime пустой

---

# 5. Кварталы и месяцы

## Created
- C_Qtr → квартал по Created
- C_Mnt → месяц по Created

## Resolved
- R_Qtr → квартал по Resolved
- R_Mnt → месяц по Resolved

---

# 6. Defect Escape Rate (DER)

## Формула
DER = Escape / Total

---

## Escape

Дефект считается escape, если:
PS links (IDs) не пустой

Это означает:
- дефект найден support
- или клиентом
- или в production

---

## Total
Все дефекты, кроме:
Status = Аннулирована

---

## Группировки

### DER by affected version
Группировка по:
Affected version

---

### DER by quarter
Группировка по: 
C_Qtr (квартал создания дефекта)

---

## DER для критичных дефектов

Считается отдельно для приоритетов:

- Major
- Critical
- Неотложный

---

# 7. Backlog

## Open defects

Дефект считается открытым на момент времени T, если:
Created <= T
AND
(Resolved is NULL OR Resolved > T)

---

# 8. Net backlog delta
Delta = Created - Resolved

---

# 9. Production defects (PS)

Дефект считается production-дефектом, если:
есть PS links

---

# 10. Фильтры

## Исключаются:
- Status = Аннулирована (для DER)

## Учитываются:
- все остальные дефекты

---

# 11. Особенности проектов

## VM / DCI6
- стандартная схема
- одинаковая логика

## BA (BILL)
- дополнительные поля:
  - Подсистема
  - Категория BILL
  - Тэги
- может иметь отличающуюся структуру

---

# 12. Ограничения

- DER не учитывает источник дефекта напрямую (используется PS как proxy)
- возможны ошибки в данных YouTrack:
  - неполные версии
  - некорректные связи
